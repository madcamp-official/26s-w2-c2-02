# Create Plan

Produce a plan that is short, testable, and ready to execute.

## Plan shape

Each step should include:

- the action
- why it matters
- the main risk
- how progress will be verified

## Rules

- Prefer the smallest plan that safely covers the work.
- Put discovery before risky implementation.
- Put fast feedback loops early.
- Call out explicit decision points when consequences are non-obvious.
- End with verification and documentation sync.

## Output

Return a numbered plan with clear checkpoints.
