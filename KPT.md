# KPT

## 2026-07-09

### Keep

- 방을 공부 모드와 게임 모드로 나눈 방향은 유지한다. 같은 화상/카메라 기반이라도 공부는 목표-집중-회고 흐름, 게임은 라운드-미션-점수 흐름으로 분리하니 UI와 상태 판단이 덜 섞인다.
- MediaPipe 분석을 로컬에서 처리하고 서버에는 상태/표정 신호만 보내는 구조를 유지한다. 개인정보 부담을 줄이면서도 집중 점수, 피로도, 산만함, 표정 미션 같은 기능을 확장할 수 있었다.
- 중앙 API가 불안정한 개발 환경에서도 로컬 데모 방으로 주요 UI와 카메라 흐름을 확인할 수 있게 한 fallback은 계속 유지한다. 시연과 디버깅 속도에 직접 도움이 된다.
- 회고 화면에 서버 summary가 없을 때도 클라이언트가 가진 마지막 집중 순위와 집중 신호 지표를 보여주는 방식은 유지한다. 방을 나가는 흐름에서도 사용자가 빈 결과 화면을 보지 않는다.
- 숨은 표정 미션에서 루미가 참가자 이름이나 정답을 직접 말하지 않고 표정 단서만 암시하는 톤은 유지한다. 게임의 긴장감과 AI 진행자 캐릭터가 잘 맞는다.

### Problem

- README가 초기 스터디룸 기획에 오래 머물러 있어서 현재 구현된 표정 게임, 집중 점수, 상세 지표, 로컬 fallback이 문서에 늦게 반영됐다.
- 공부 모드와 게임 모드가 같은 StudyRoom 화면을 공유하면서 상태 이름, 점수 의미, 카메라 판정 기준이 섞일 위험이 있다. 새 기능을 추가할수록 모드별 조건문이 커질 수 있다.
- 집중 감지 기준은 실제 사용자 자세에 민감하다. 고개 돌림, 하품, 시선 이탈, 자세 흔들림은 민감도를 낮추면 놓치고 높이면 오탐이 늘어난다.
- 숨은 표정 미션의 지목, 방해 카드, 라운드 전환, 로컬 fallback이 얽히면서 경계 타이밍 버그가 생기기 쉽다.
- 게임 모드 일부는 선택 UI에 보이지만 아직 다음 업데이트 안내 상태다. 사용자가 가능한 게임과 준비 중인 게임을 명확히 구분할 수 있어야 한다.

### Try

- 공부 모드와 게임 모드의 상태 계산을 더 명확한 모듈로 나눈다. 화면은 공유하더라도 scoring, detection rule, room action은 모드별로 분리한다.
- 숨은 표정 미션의 라운드 전환, 재입장, 지목 성공/실패, 방해 카드 해제 조건을 테스트 케이스로 고정한다.
- 집중 감지 기준은 판정 테스트 화면에서 실제 사용자 샘플을 더 모아 기본값을 조정한다. 특히 시선 이탈 지속 시간, 하품 지속 시간, 자세 흔들림 기준은 오탐 로그를 보고 다듬는다.
- README에는 안정적인 제품 개요만 두고, 세부 구현 변경은 CHANGELOG와 docs 문서에 먼저 쌓은 뒤 발표 전 한 번에 요약 반영한다.
- 카피캣 릴레이와 포커페이스 블러프는 UI 선택 가능 상태와 실제 구현 완료 상태가 어긋나지 않게 단계별 feature flag 또는 disabled 안내를 정리한다.
