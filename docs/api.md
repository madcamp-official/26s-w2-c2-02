# Roomi API Draft

## REST

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/health` | Check API availability. |
| `POST` | `/rooms` | Create a room with a host participant. |
| `POST` | `/rooms/join` | Join an existing room by invite code. |
| `GET` | `/rooms/:inviteCode` | Read a room snapshot by invite code. |

## Socket.IO Events

Client events are defined in `packages/shared/src/realtime-events.ts`.

| Event | Direction | Purpose |
|---|---|---|
| `room:join` | client to server | Join a realtime room and receive a snapshot. |
| `room:leave` | client to server | Leave the realtime room channel. |
| `participant:update-status` | client to server | Publish focus/break/away status updates. |
| `room:updated` | server to client | Broadcast the latest room snapshot. |
| `roomi:message` | server to client | Send an operator message generated by Roomi. |
| `error` | server to client | Report a recoverable realtime error. |

## Environment

Copy `.env.example` to `.env` before running local services.

| Variable | Purpose |
|---|---|
| `API_PORT` | Backend HTTP and Socket.IO port. |
| `CLIENT_ORIGIN` | Allowed desktop renderer dev origin. |
| `DAILY_API_KEY` | Daily API key for room/token creation. |
| `DAILY_DOMAIN` | Daily domain used by the video provider. |
| `OPENAI_API_KEY` | LLM provider API key, kept server-side only. |
