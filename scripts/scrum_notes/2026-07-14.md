### **오늘 할 일**

**A. ML 집중도 판정 검증**

- A1. 실제 ML API 서버를 `VITE_ROOMI_ML_API_URL`로 연결해 `/v1/focus/predict` 요청/응답 흐름 확인
- A2. Rule-Based 판정과 ML 서버 판정을 같은 webcam 입력으로 비교해 label과 score 차이 기록
- A3. 20초 feature window 크기와 Rule-Based 기준값이 데모 흐름에 맞는지 조정
- A4. ML 서버 연결 실패, CORS 오류, 라우팅 오류 상황에서 fallback과 안내 문구가 자연스럽게 보이는지 확인

### **궁금한/필요한/알아낸 것**

- MediaPipe 테스트 화면을 사용하려면 `pnpm install`로 `@mediapipe/tasks-vision` dependency 설치 필요
- ML 서버를 붙일 때 desktop renderer의 `VITE_ROOMI_ML_API_URL` 값과 서버 CORS 허용 origin 확인 필요
- ML 서버가 없거나 실패해도 Rule-Based 판정이 계속 표시되므로 데모 자체는 fallback으로 진행 가능
